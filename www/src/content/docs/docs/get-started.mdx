Fiberplane is a [Hono](https://hono.dev) middleware that provides a built-in Playground UI for exploring and testing Hono service APIs. It leverages [OpenAPI](https://www.openapis.org/what-is-openapi) to deliver interactive API documentation, allowing users to send requests with generated payloads through the UI.

<Steps>
1. ### Set up a Hono project

    You can set up a basic Hono project via your terminal:

    <PackageManagers type="create" pkg="hono@latest" args="my-app" />

    If you want a project that includes an ORM, a database, the option to include Hono Zod OpenAPI (see step 2), Fiberplane middleware (see step 3) and is ready to deploy as a Cloudflare Worker, we recommend using [HONC ü™ø](https://honc.dev/):

    <PackageManagers type="create" pkg="honc-app@latest" />

2.  ### Add OpenAPI spec to your Hono application

    Fiberplane Playground Hono middleware works with an OpenAPI spec. There are two main approaches to obtain one:

    #### A) Start with OpenAPI from the Beginning

    Designing an API with documentation in mind from the start ensures a well-structured approach. Use [Zod OpenAPI Hono](https://github.com/honojs/middleware/tree/main/packages/zod-openapi) or from [Rhinobase the hono-openapi package](https://github.com/rhinobase/hono-openapi) to define the API while maintaining clear documentation.
    Besides documentation, Zod OpenAPI Hono and Rhinobase Hono-OpenAPI also support input validation and can ensure typesafety.

    - ##### Zod OpenAPI Hono

      Install the package alongside hono and zod:

      {" "}
      <PackageManagers type="add" pkg="hono zod @hono/zod-openapi" />

      Create an app using Hono Zod

      ```ts
      import { OpenAPIHono } from "@hono/zod-openapi";

      const app = new OpenAPIHono<{
        Bindings: Bindings;
        Variables: Variables;
      }>();
      ```

      Define your input schema's in Zod:

      ```ts
      import { z } from "@hono/zod-openapi";

      const NewUserSchema = z
        .object({
          name: z.string().openapi({
            example: "Matthew"
          }),
          email: z.string().email().openapi({
            example: "matthew@cloudflare.com"
          })
        })
        .openapi("NewUser");
      ```

      Define your routes and endpoint in Zod using the input schemas:

      ```ts
      import { createRoute } from "@hono/zod-openapi";

      const getUsers = createRoute({
        method: "get",
        path: "/api/users",
        responses: {
          200: {
            content: { "application/json": { schema: z.array(UserSchema) } },
            description: "Users fetched successfully"
          }
        }
      });
      ```

      Set up the application and define the business logic for the handlers:

      ```ts
      app.openapi(getUsers, async (c) => {
        const db = c.get("db");
        const users = await db.select().from(schema.users);
        return c.json(users);
      });
      ```

      Make the OpenAPI spec available:

      ```ts
      app.doc("/openapi.json", {
        openapi: "3.0.0",
        info: {
          title: "D1 Honc! ü™ø‚òÅÔ∏è",
          version: "1.0.0",
          description: "D1 Honc! ü™ø‚òÅÔ∏è"
        }
      });
      ```

    - ##### Rhinobase Hono-OpenAPI
      Besides Zod Rhinobase Hono-OpenAPI supports multiple validation schemas.
      {" "}
      <PackageManagers
        type="add"
        pkg="hono-openapi @hono/zod-validator zod zod-openapi"

    #### B) Add OpenAPI to an Existing Codebase

    The other option is to integrate OpenAPI documentation into an existing Hono codebase. Refactoring the application to use Hono Zod OpenAPI or Rhinobase provides fine-grained documentation, offering long-term benefits, but requires more effort. For a quicker solution, Fiberplane provides a helper function to generate an OpenAPI spec from the existing code.

    <PackageManagers type="add" pkg="@fiberplane/hono" />

    This step involves importing the `createOpenAPISpec` function from Fiberplane and creating an endpoint at `/openapi.json` to generate and return an OpenAPI spec.

    ```ts
    import { createOpenAPISpec } from "@fiberplane/hono";

    app.get("/openapi.json", (c) => {
      const spec = createOpenAPISpec(app, {
        info: { title: "My API", version: "1.0.0" }
      });
      return c.json(spec);
    });
    ```

3.  ### Add Fiberplane Middleware

    With the OpenAPI spec available, the Fiberplane middleware can now be installed and pointed to the OpenAPI spec. <PackageManagers type="add" pkg="@fiberplane/hono" />

    ```ts
    import { createFiberplane } from "@fiberplane/hono";

    app.use(
      "/fp/*",
      createFiberplane({
        openapi: { url: "/openapi.json" }
      })
    );
    ```

    After starting the application, Fiberplane's native UI will be available at the `/fp` endpoint of the Hono application.
    For example, if the Hono app is running on `http://localhost:8787`, the Fiberplane Playground will be accessible at `http://localhost:8787/fp/`.
